# 🛡️ SOC Resources

Welcome to the **SOC Resources** repository. This project is designed to be a curated roadmap for anyone looking to enter the field of Security Operations or for existing analysts looking to sharpen their skills.

The goal is to move from **Foundational Knowledge** $\rightarrow$ **Tool Proficiency** $\rightarrow$ **Analytical Mindset** $\rightarrow$ **Professional Expertise**.

---

## 🗺️ The SOC Analyst Roadmap

### 🟢 Level 1: The Foundations
*Before touching a SIEM, you must understand how the "plumbing" of the internet works.*
- [ ] **Networking**: OSI Model, TCP/UDP, DNS, DHCP, HTTP/S, ICMP.
- [ ] **Operating Systems**: Windows Event Logs, Linux Syslog, Bash/PowerShell basics.
- [ ] **Security Basics**: CIA Triad, Common Ports, Encryption vs. Hashing.

### 🟡 Level 2: The Toolbelt
*Learning the tools used to detect and respond to threats.*
- [ ] **SIEM (Security Information & Event Management)**: Log aggregation, querying, and alerting.
- [ ] **EDR (Endpoint Detection & Response)**: Process trees, telemetry, and isolation.
- [ ] **Traffic Analysis**: Packet capture (PCAP) analysis and flow data.
- [ ] **Threat Intelligence**: Using OSINT, MISP, and AlienVault OTX.

### 🟠 Level 3: Analysis & Frameworks
*Moving from "seeing an alert" to "understanding an attack".*
- [ ] **MITRE ATT&CK**: Mapping adversary behavior to the matrix.
- [ ] **Incident Response**: The NIST/SANS IR lifecycle (Preparation $\rightarrow$ Recovery).
- [ ] **Log Analysis**: Learning to spot anomalies in Windows Event IDs and Web Server logs.

### 🔴 Level 4: Advanced Operations
*Proactive defense and optimization.*
- [ ] **Threat Hunting**: Hypothesis-based searching for undetected threats.
- [ ] **Detection Engineering**: Writing Sigma/YARA rules to catch new threats.
- [ ] **Malware Analysis**: Basic static and dynamic analysis.
