# Implementation Plan: soc-resources Repository

## Project Goal
To create a comprehensive, curated, and community-driven repository that serves as a roadmap and resource hub for aspiring and professional Security Operations Center (SOC) Analysts.

---

## Phase 1: Foundation & Structure (Current)
- [x] Define the target audience (Beginners, Junior Analysts, Career Switchers).
- [x] Establish the repository directory structure.
- [x] Create a professional `README.md` that acts as the landing page and primary roadmap.
- [x] Create an implementation plan to track progress.

## Phase 2: Resource Curation (The "Meat")
- **Foundational Knowledge**: Curate links for Networking (TCP/IP, DNS, HTTP), OS Internals (Windows/Linux), and Security Basics.
- **Tooling Mastery**: 
    - SIEM (Splunk, ELK, Microsoft Sentinel).
    - EDR/XDR (CrowdStrike, SentinelOne, Microsoft Defender).
    - Traffic Analysis (Wireshark, Zeek, Brim/Zui).
- **Frameworks & Methodologies**: Detailed guides on MITRE ATT&CK, NIST Incident Response, and the Diamond Model.

## Phase 3: Hands-on Application (The "Lab")
- **Home Lab Guide**: Create a "Starter Kit" for building a SOC lab (VirtualBox, pfSense, Security Onion, etc.).
- **Practical Platforms**: Curated list of rooms/paths on TryHackMe, HackTheBox, and LetsDefend.
- **Dataset Sources**: Links to PCAPs and logs for practice analysis.

## Phase 4: Professional Development
- **Certification Roadmap**: Tiered recommendations (e.g., Security+ $\rightarrow$ CySA+ $\rightarrow$ BTL1 $\rightarrow$ GCIH).
- **Interview Prep**: Common SOC interview questions and "Day in the Life" expectations.
- **Soft Skills**: Documentation templates, reporting guidelines, and communication tips.

## Phase 5: Growth & Maintenance
- **Contribution Guide**: `CONTRIBUTING.md` to allow the community to add resources.
- **Issue Tracking**: Using GitHub Issues to request specific resources or report dead links.
- **Versioning**: Regular updates to ensure tools and links remain current.
